#1. OpenClaw on Ubuntu VM (UTM on Macbook Air/M3)

  #a. (Solved): M3 is ARM, and there is no official Chrome for Ubuntu/ARM. Firefox doesn't work with OpenClaw.
Tried alternatives like: Chromium and Brave.
There is some awkwardness in this setup where the browser window opened by OpenClaw is unable to download files. It was quite a task to get "Relay" to work as an extension, and force OpenClaw to reuse browser window opened by me.

  #b. (Solved): On Mac UTM, set the VM to use a different MAC and get its own IP address. Don't reuse Mac's internet connection. There was some trouble for UTM to reuse Mac's IP properly in IPv6 format, and blocked Telegram from working properly.

  #c. (Solved): I keep running into ratelimit and cooldown with Anthropic and Google AI Studio.
DeepSeek is running well. I haven't tried OpenAI and others yet.

  #d. (Not fixed): Making dashboard visible on LAN. It requires config like this -
    "mode": "remote",
    "bind": "lan",
    "tls": {
      "enabled": true,
      "autoGenerate": true
    },
    "remote": {
      "url": "wss://192.168.1.XX:18789" 
    }
But even after all this RPC Probe was still trying 127.0.0.1 that fails.

#2. Github on UTM VM
  #a. Login was a challenge using Firefox / Chromium. There's circus involved as "gh auth" fails too soon. The trick is to start one cycle but not click submit on the browser. Then restart it on CLI, and sychronize second key of CLI with first submit of the browser. Then it works.

#3. UTM VM
UTM Ubuntu on Mac doesn't auto-sync time. Tricks are to install QEMU agent as explained by Gemini.

#4. Google web_search
Add this: 
"tools": {
    "web": {
      "search": {
        "enabled": true,
        "provider": "gemini",
        "gemini": {
          "apiKey": "deleted",
          "model": "gemini-2.5-flash"
        }
      },
Don't need to add "plugins" section as shown in older results.
